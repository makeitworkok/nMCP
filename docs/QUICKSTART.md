# Quickstart Guide

Get the nMCP Module running on a live station and talking to a client in under 15 minutes.

---

## Prerequisites

| Requirement | Version |
|---|---|
| Java | 8 or 11 |
| Gradle | 7.6.4 (wrapper included) |
| Python | 3.9+ |
| Niagara | Station prior to version 4.13, with HTTP enabled |
| Niagara module verification | `low` (see below) |

### Set module verification to `low`

For development/community modules, set this in Niagara defaults:

```
C:\Niagara\Niagara-<version>\defaults\system.properties
```

Add or update:

```
niagara.moduleVerificationMode=low
```

What this means:

- `low` = warnings only (no enforcement), which is the development escape hatch.
- If the blacklist entry for this property is commented out/inactive, the setting takes effect.
- This applies to both station process behavior and Workbench behavior.

Restart the station after changing this property.

### Enable MCP write mode (optional)

Set BMcpService `Read Only` to `false` in Workbench and save. This slot persists across restarts.

- `Read Only = true`: write-capable tools are blocked.
- `Read Only = false`: write-capable tools can execute, subject to allowlist checks and tool-level validation.
- Return this selector to `true` after write operations are complete.

## Step 1 — Build the module

```powershell
cd C:\nMCP
java -jar gradle\wrapper\gradle-wrapper.jar clean jar
```

Output: `build/libs/nMCP.jar`

---

## Step 2 — Install to the station

Copy the jar to all active Niagara module locations:

```powershell
copy build\libs\nMCP.jar "C:\Niagara\Niagara-<version>\modules\nMCP.jar"
copy build\libs\nMCP.jar "C:\Niagara\Niagara-<version>\sw\0.8.0\nMCP.jar"
```

> Overwrite any existing `nMCP.jar`. Back up the old one first if needed.

If `sw\0.8.0` does not exist yet, create it first.

---

## Step 3 — Add BMcpService to the station

1. Restart the station (or reload modules in Niagara Workbench).
2. In Workbench, navigate to **Config → Services**.
3. Open the `nMCP` palette.
4. Drag **BMcpService** into **Services**.
5. Save and start the service.

The endpoint is now live at `http://<station-host>/nmcp`.

**Verify the service is running** — open a browser or curl to `GET /nmcp`:

```bash
curl http://localhost/nmcp
```

Expected response (JSON with your auto-generated token):

```json
{
  "service": "nMCP",
  "version": "0.8.0",
  "endpoint": "/nmcp",
  "token": "<auto-generated-token>",
  "note": "Copy this token for your MCP client X-MCP-Token header"
}
```

Expected station startup logs for v0.8.0:

- `BMcpService: detected Niagara platform version <x.y.z>`
- On Niagara 4.13+: `EULA of the version 4.13 and greater forbids use of AI, see Section 3.1(q) for details.`
- If version detection fails: `BMcpService: Niagara platform version could not be detected; EULA check could not be confirmed.`

## Step 4 — Start the Python auth proxy

Niagara uses web authentication. Run a local proxy that handles this
automatically so clients don't need to implement it.

```bash
# Recommended: follow the companion client guide
# https://github.com/makeitworkok/nMCP-client#readme
<start-your-proxy-command>
```

| Argument | Required | Description |
|---|---|---|
| `--niagara-url` | Yes | Base URL of the Niagara station (e.g., `https://localhost`) |
| `--niagara-user` | Yes | Niagara web username |
| `--niagara-pass` | Yes | Niagara web password |
| `--token` | No | Optional client token; if omitted, all requests are accepted without auth |
| `--backend-mcp-token` | No | If omitted, auto-discovered from `GET /nmcp` after login |
| `--port` | No (default: `8765`) | Port the proxy listens on |

The proxy logs its session state to stdout. A successful start looks like:

```
[mcp-proxy] Connecting to Niagara at https://localhost …
[mcp-proxy] Auto-discovered backend MCP token from station
[mcp-proxy] Authenticated as 'mcp'
[mcp-proxy] Listening on http://127.0.0.1:8765/nmcp
```

## Step 5 — Verify end-to-end

```bash
curl -X POST http://127.0.0.1:8765/nmcp \
  -H "X-MCP-Token: <your-client-token>" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}'
```

You should receive a `tools` array listing all 36 available tools.

At minimum, verify these v0.6.1 wiresheet tools are present:
- `nmcp.wiresheet.plan`
- `nmcp.wiresheet.diff`
- `nmcp.wiresheet.apply`
- `nmcp.wiresheet.links`

## Step 6 — Connect a client

### nMCP-client (recommended)

Use the lightweight companion client for scripted calls and app integration:
- Repository: [nMCP-client](https://github.com/makeitworkok/nMCP-client)
- README: [nMCP-client README](https://github.com/makeitworkok/nMCP-client#readme)
- Examples: [nMCP-client examples](https://github.com/makeitworkok/nMCP-client/tree/main/examples)

Typical use:
1. Keep this module (`nMCP`) running on the station.
2. Run the auth proxy from this repo.
3. Use `nMCP-client` to call `tools/list` and `tools/call` against `http://127.0.0.1:8765/nmcp`.

### Claude Desktop

Add to your `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "niagara": {
      "url": "http://127.0.0.1:8765/nmcp",
      "headers": {
        "X-MCP-Token": "<your-client-token>"
      }
    }
  }
}
```

### VS Code Copilot

Add to your workspace or user settings:

```json
"mcp.servers": {
  "niagara": {
    "url": "http://127.0.0.1:8765/nmcp",
    "headers": {
      "X-MCP-Token": "<your-client-token>"
    }
  }
}
```

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `GET /nmcp` redirects to `/login` | Auth required, no proxy | Start your local web-auth proxy first |
| `NoSuchMethodError` on tool call | Stub/runtime API mismatch | Check Niagara version matches stub signatures |
| `Found interface javax.baja.sys.Cursor, but class was expected` | Old module still loaded | Re-copy jar to all target locations and restart station |
| Module loads with warnings only | Verification mode not set | Set `niagara.moduleVerificationMode=low` in defaults `system.properties` |
| `LinkageError` or `IllegalAccessError` | Module conflict | Ensure only one `nMCP.jar` is in the modules directory |
| Proxy reports auth failure | Wrong credentials or auth handshake issue | Check `--niagara-user` and `--niagara-pass`; ensure HTTP (not HTTPS) |
| Tool returns `AlarmService not available` | AlarmService not configured | Add/start AlarmService in station Services |
